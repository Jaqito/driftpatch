# DriftPatch

Generic upstream-drift engine. Turns upstream changes (changelogs, docs, specs) into reviewable, validated, repo-aware code updates.

## Architecture

```
Generic Engine + Provider Adapter + Repo Skill = useful automated PRs
```

- **Engine** — reusable across every repo and provider; owns the pipeline.
- **Provider Adapter** — per-upstream, FDE-authored, reused across customer engagements.
- **Repo Skill** — per-repo, generated by `driftpatch init`, refined by humans.

## Packages

| Package | Purpose |
| --- | --- |
| `@driftpatch/core` | Engine, types, pipeline orchestration |
| `@driftpatch/cli` | `driftpatch` command-line interface |
| `@driftpatch/adapter-sdk` | `ProviderAdapter` interface and helpers |
| `@driftpatch/adapter-generic` | Fallback markdown adapter (only adapter that ships) |
| `@driftpatch/eval` | Grader and fixture harness |

`examples/adapter-polaris` is a reference adapter authored against `adapter-sdk` only — same constraints a real FDE would have. Not part of the shipped tool.
